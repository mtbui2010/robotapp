## What it does

RobotApp is the **UI tier** of a three-mode robot stack — UI, CLI, and Python API
all share one runtime ([robot_agent](/robotapp/stack/robot-agent/)). The
dashboard turns any [robot_agent](/robotapp/stack/robot-agent/)-based robot
(reference: [kcare_robot](/robotapp/stack/kcare-robot/), an assistive 6-DOF
mobile manipulator) into a network-addressable agent — driveable from a
browser on the other side of the world, deployed globally on Cloudflare Pages.

<CardGrid>
  <Card title="Live RGB-D streaming" icon="rocket">
    WebSocket bridges to RealSense D405 and Femto Bolt RGB-D cameras.
    `zlib`-compressed `uint16` depth is decoded in the browser and rendered
    with a JET colormap. Pixel-hover gives a live millimetre readout.
  </Card>
  <Card title="Open-vocabulary perception" icon="magnifier">
    Type <code>apple</code> or <code>the red mug on the table</code>;
    the backend routes to a TCP VLM service (GroundingDINO + GroundedSAM +
    mask2grasps) and lifts detections to 6-DOF grasp poses.
  </Card>
  <Card title="Streaming task plans" icon="list-format">
    A single WebSocket carries the full lifecycle:
    <code>start → plan → step_start → step_log → step_done → done</code>.
    Inline frames render next to the step that produced them.
  </Card>
  <Card title="Hot-swappable skills" icon="setting">
    Register new ROS service / topic / action / WebRTC / TCP / LLM clients
    from the UI. Reload skills with one POST — no robot restart.
  </Card>
  <Card title="Three execution modes" icon="puzzle">
    Same skill, three entry points: <code>POST /skill/&lt;name&gt;</code>,
    <code>kcare_robot pick::apple</code>, or
    <code>from kcare_robot.skills.pick import pick</code>.
  </Card>
  <Card title="Edge deployment" icon="cloud-download">
    Static export, served from Cloudflare Pages with a custom domain
    (<a href="https://robot.aistations.org">robot.aistations.org</a>) — sub-100 ms TTFB anywhere.
  </Card>
</CardGrid>

## A guided tour of the dashboard

<Image src={dashboardFull} alt="Full dashboard screenshot" />

The interface is split into four functional zones. Click through them below.

<Tabs>
  <TabItem label="Camera feed">
    <Image src={featureCamera} alt="Live RGB / depth camera tabs with FPS counter and capture controls" />
    Multi-camera tabs (head_rgb, head_depth, arm_rgb, arm_depth, log_image)
    persist their order in <code>localStorage</code>. Drag to reorder. RGB
    streams as JPEG; depth streams as raw <code>uint16</code> +
    <code>zlib</code>, decoded and colorised client-side. Rectangle-draw
    overlay is wired to detector inputs.
  </TabItem>
  <TabItem label="Unified agent">
    <Image src={featureAgent} alt="Structured / unstructured agent prompt with language toggle" />
    Two prompt modes — <strong>Structured</strong> for direct skill calls
    (<code>moveh::down</code> + key=value params) and
    <strong>Unstructured</strong> for natural-language tasks routed through
    an LLM planner. Language picker (EN / KO / VI). Ctrl+Enter to dispatch.
  </TabItem>
  <TabItem label="Task plan">
    <Image src={featurePlan} alt="Plan + execution panel with step results" />
    The LLM plan and step-by-step execution timeline. Each step shows
    status (<code>✓ done</code> / <code>✗ failed</code> / <code>⟳ running</code>),
    a JSON result, and any log image emitted mid-skill — all streamed over
    one WebSocket.
  </TabItem>
  <TabItem label="Skill registry">
    <Image src={featureSkills} alt="Skill registry grouped by category" />
    Live list of every registered skill, grouped by category
    (recognition, pick, place, mobile, arm, head, lift…). CRUD on each
    skill, JSON-edit per-skill configs, hot-reload via
    <code>POST /skills/reload</code>.
  </TabItem>
  <TabItem label="Connections">
    <Image src={featureConn} alt="Device connections panel with ping all and ROS scan" />
    Every ROS service / topic / action / WebRTC / TCP / LLM client
    registered with the robot. <strong>Ping All</strong> health-checks the
    bus; <strong>Scan ROS</strong> discovers nodes via
    <code>get_ros2_node_names_and_types()</code>.
  </TabItem>
</Tabs>

## Architecture at a glance

```text
                ┌──────────────────────────────────────┐
                │  robotapp   ·  Next.js 14 + Tailwind │   browser / Cloudflare Pages
                │  WebSocket + REST client (TypeScript)│
                └────────────────┬─────────────────────┘
                                 │  HTTP + WS
                                 ▼
       ┌──────────────────────────────────────────────────┐
       │  robot_agent   ·   FastAPI                       │   robot host · port 8001
       │  · SkillRegistry · DeviceManager · UnifiedAgent  │
       │  · /ws/camera/{id}   /ws/agent   30+ REST        │
       └────────────┬──────────────────────┬──────────────┘
                    │                      │
                    ▼                      ▼
        ROS2 Humble · rclpy · Nav2     Devices · RealSense D405
        joint_states · /navigate_to_   Femto Bolt · KAAIR cobot
        pose · /kaair_worker/*         TCP VLM service (GPU)
```
